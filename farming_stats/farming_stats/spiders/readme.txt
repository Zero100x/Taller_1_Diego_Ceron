3. Cuadro Comparativo Scrapy vs Selenium
| Característica   | **Scrapy**                                                   | **Selenium**                                                  |
| ---------------- | ------------------------------------------------------------ | ------------------------------------------------------------- |
| **Ventaja 1**    | Muy rápido y eficiente para scraping masivo.                 | Permite interactuar con sitios dinámicos (JavaScript).        |
| **Ventaja 2**    | Estructura orientada a proyectos, ideal para grandes ETL.    | Simula un navegador real (clics, scroll, formularios).        |
| **Ventaja 3**    | Maneja datos estructurados fácilmente y exporta a CSV/JSON.  | Compatible con descargas automáticas y pruebas automatizadas. |
| **Desventaja 1** | Difícil con páginas dinámicas (JS).                          | Mucho más lento que Scrapy.                                   |
| **Desventaja 2** | Requiere mayor curva de aprendizaje.                         | Alto consumo de recursos (RAM y CPU).                         |
| **Desventaja 3** | No permite automatizar descargas ni interacciones complejas. | El código suele ser más frágil ante cambios en el DOM.        |
